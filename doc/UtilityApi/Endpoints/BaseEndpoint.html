Overview

Common API methods supported for all endpoints. This class is to be subclassed, not instantiated directly. Supports performing all the methods asynchronously.

Direct Known Subclasses

- (Object) modify(uid, options)

Modify an object using the provided options. This is commonly used with accounts when utility login credentials need to be updated, or with services when [de]activating the service. By default, this will trigger all services related to this object to update their bill and interval datasets.

Parameters:

  • uid —

    The String with an object uid.

  • options —

    The options object with a corresponding type (e.g. AccountOptions for accounts).

Returns:

  • the modified object as an instance of the corresponding model.

Raises:

  • Errors::NotFound if the uid is not found (HTTP 404).

  • Errors::BadRequest if the options are malformed (HTTP 400).

  • Errors::BaseError or subclass if an HTTP error code received.

def modify(uid, options)
  body = @connection.post("#{uid}/modify", options).body

  if body.count == 1
    # XXX: it is not documented in api
    self.class::MODEL.new(body.values.first)
  else
    self.class::MODEL.new(body)
  end
end
